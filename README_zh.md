## 🚀 为什么写这本书？

三个月前，我的 Agent 团队崩溃了。四个 Agent 同时抢一个任务，记忆系统把昨天的结论和今天的需求搅成一团，监察员像个摆设一样看着它们互相甩锅。我盯着屏幕，感觉自己不像个老板，更像一个被"数字实习生"包围的救火队长。

那次翻车，比我看过的任何教程都管用。所以我把教训写了下来。

**管理的本质，正从"管人"转向"管 Agent"。** 这本书就是这段转型的实战手册——用故事讲方法，用真实案例撑腰，每章都给今天就能上手的做法。没有吹水，没有魔法，只有踩坑踩出来的经验。

## 📚 你会学到什么

- 如何在 **21 章内从零搭起一支能运转的 Agent 团队**
- 写出**真正管用的 System Prompt**——它不是约束，而是注意力分配策略
- 设计**可扩展的分工体系**，避免一扩大就崩
- 构建**记忆系统**与**沟通协议**，让 Agent 不再各说各话
- 建立**监察员制度**，让 Agent 管理 Agent
- 从"等任务"进阶到**目标导向、主动提案**的 Agent
- **成本控制**，别让 API 账单爆掉
- **翻车生存指南**：Agent 搞砸的真实复盘，以及如何重建
- **自进化与自我修复**：越跑越聪明的团队

## 🎯 这本书适合谁

- **创业者 / 业务负责人**：有想法，但实现要依赖程序员
- **技术负责人**：被人类团队的协作损耗淹没
- **爱折腾的运营者**：想要 7×24 覆盖，又不想 7×24 值班
- **任何人**：用过单个 Agent，然后想"这还能更强"的人

> 你不需要是 Prompt 工程大神。你需要的是用 **Agent 的 CEO** 思维，而不是程序员思维。

## 🗺️ 21 天路线：从 L0 到 L3

行业对 Agent 团队成熟度已有共识。本书带你从零走到"一人监督的多 Agent 团队"：

| 层级 | 含义 | 你将抵达 |
|------|------|----------|
| **L0** | 还没有 Agent 团队 | ← *你从这里出发* |
| **L1** | 单 Agent，固定自动化 | 第 04 章 |
| **L2** | 多 Agent，手动协调 | 第 06–08 章 |
| **L3** | Supervisor-Worker，一人监督 | 第 10、17、18 章 ← **本书目标** |
| **L4 / L5** | Agent 舰队 + 自组织 | 你后续的进化方向 |
